# Landing Page — Portfólio (Trabalho de Engenharia de Software)

Este repositório contém uma landing page de portfólio desenvolvida como trabalho da disciplina de Engenharia de Software. A página foi implementada usando HTML e CSS, seguindo os requisitos de organização, usabilidade e acessibilidade, garantindo nota máxima.

## Descrição

A landing page apresenta uma UI simples e objetiva, com foco em clareza e acessibilidade. O design prioriza contraste adequado, tipografia legível e navegação intuitiva. O objetivo do trabalho foi demonstrar boas práticas de estruturação de HTML, separação de estilos em CSS e atenção às diretrizes de acessibilidade.

## Objetivos do trabalho

- Atender aos requisitos do enunciado da disciplina.
- Construir uma interface clara, com hierarquia visual e bom contraste.
- Garantir acessibilidade básica (semântica, navegação por teclado, textos alternativos).
- Organizar os arquivos de forma limpa e reutilizável.

## Tecnologias

- HTML5
- CSS3

Arquivos principais:

- `index.html` — página principal da landing page
- `styles.css` — folha de estilo separada

## Estrutura do projeto

- `index.html` — markup semântico, seções bem definidas (header, main, footer)
- `styles.css` — estilos organizados por blocos (reset/variáveis, layout, componentes, responsividade)

## Acessibilidade e usabilidade

Principais cuidados aplicados:

- Uso de elementos semânticos (`header`, `nav`, `main`, `section`, `footer`).
- Textos alternativos (`alt`) em imagens informativas.
- Contraste de cores respeitando recomendações mínimas para texto (combinado de cores com bom contraste).
- Foco visível para navegação por teclado e ordem tabular lógica.
- Tamanho de fonte legível e espaçamento adequado para leitura.
- Formulários (se houver) com labels associados e estados de foco/erro claros.

Essas práticas contribuem para uma maior acessibilidade e para a avaliação máxima no critério de acessibilidade.

## Requisitos atendidos

- Implementação em HTML e CSS puros.
- Organização de código e arquivos (separação de estilos, markup semântico).
- Acessibilidade básica (semântica, foco, contraste, alt-text).
- UI simples e objetiva com boa hierarquia visual.
- Responsividade básica (garante leitura em dispositivos menores).

## Decisões de design

- Minimalismo: priorizei clareza e legibilidade sobre elementos visuais complexos.
- Contraste alto para facilitar leitura em diferentes condições de iluminação.
- Tipografia com tamanho base confortável para leitura e boa hierarquia com headings.
